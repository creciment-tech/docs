## Overview

Create a delivery using a `tempDeliveryId` obtained from the pricing endpoint. This converts the price estimate into an actual delivery that can be paid for and dispatched.

<Note>
  **Required permission:** `delivery:create`
</Note>

## Endpoint

```
POST /api/deliveries/create
```

## Request

### Headers

<ParamField header="X-API-Key" type="string" required>
  Your Chidori API key
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

### Body parameters

<ParamField body="tempDeliveryId" type="string" required>
  The temporary delivery ID from the pricing response
</ParamField>

<ParamField body="deliveryName" type="string">
  A descriptive name for the delivery (e.g., "Office Supplies")
</ParamField>

<ParamField body="deliveryType" type="string">
  Type of delivery. Options: `SendDelivery` or `RequestPickup`
</ParamField>

<ParamField body="receiverPhoneNumber" type="string">
  Phone number of the recipient (e.g., `+2348012345678`)
</ParamField>

<ParamField body="senderPhoneNumber" type="string">
  Phone number of the sender (e.g., `+2348087654321`)
</ParamField>

<ParamField body="additionalNotes" type="string">
  Any special instructions for the driver
</ParamField>

<ParamField body="scheduledTime" type="string">
  ISO 8601 datetime for scheduled deliveries (e.g., `2025-01-15T10:00:00.000Z`)
</ParamField>

## Response

### Success response

<ResponseField name="status" type="boolean">
  Always `true` for successful requests
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Unique delivery ID (UUID)
    </ResponseField>
    <ResponseField name="trackingId" type="string">
      Human-readable tracking ID (e.g., `CHI-ABC123`)
    </ResponseField>
    <ResponseField name="deliveryStatus" type="string">
      Current status of the delivery (starts as `PENDING`)
    </ResponseField>
    <ResponseField name="bulk" type="string">
      Bulk ID if this delivery is part of a bulk (null otherwise)
    </ResponseField>
    <ResponseField name="order" type="number">
      Order position if part of a bulk delivery
    </ResponseField>
  </Expandable>
</ResponseField>

## Delivery workflow

<Steps>
  <Step title="Get price estimate">
    Call `/api/pricing/single` to get a `tempDeliveryId`
  </Step>
  <Step title="Create delivery">
    Use this endpoint with the `tempDeliveryId`
  </Step>
  <Step title="Pay for delivery">
    Call `/api/payments/charge` with the delivery ID
  </Step>
  <Step title="Track delivery">
    Use webhooks or WebSocket for real-time updates
  </Step>
</Steps>

## Delivery statuses

| Status | Description |
|--------|-------------|
| `PENDING` | Delivery created, awaiting payment |
| `PAID` | Payment received, awaiting driver assignment |
| `ASSIGNED` | Driver assigned to the delivery |
| `PICKED_UP` | Driver has picked up the package |
| `IN_TRANSIT` | Package is on the way to destination |
| `DELIVERED` | Package successfully delivered |
| `CANCELLED` | Delivery was cancelled |

<Tip>
  The `tempDeliveryId` expires after a certain period. If you receive a "not found" error, request a new price estimate.
</Tip>
