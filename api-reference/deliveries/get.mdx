## Overview

Retrieve detailed information about a specific delivery using either its `id` (UUID) or `trackingId`.

<Note>
  **Required permission:** `delivery:read`
</Note>

## Endpoint

```
GET /api/deliveries/get
```

## Request

### Headers

<ParamField header="X-API-Key" type="string" required>
  Your Chidori API key
</ParamField>

### Query parameters

Provide either `id` OR `trackingId`:

<ParamField query="id" type="string">
  The unique delivery ID (UUID)
</ParamField>

<ParamField query="trackingId" type="string">
  The human-readable tracking ID (e.g., `CHI-ABC123`)
</ParamField>

## Response

### Success response

<ResponseField name="status" type="boolean">
  Always `true` for successful requests
</ResponseField>

<ResponseField name="data" type="object">
  Complete delivery object with all details including:
  - Delivery ID and tracking ID
  - Status and timestamps
  - Pickup and destination coordinates
  - Pricing information
  - Driver details (when assigned)
  - Payment status
</ResponseField>

## Examples

<RequestExample>
```bash cURL (by ID)
curl "https://api.chidori.africa/enterprise-service/api/deliveries/get?id=abc123-def456-ghi789" \
  -H "X-API-Key: sk_live_xxx.yyy"
```

```bash cURL (by trackingId)
curl "https://api.chidori.africa/enterprise-service/api/deliveries/get?trackingId=CHI-ABC123" \
  -H "X-API-Key: sk_live_xxx.yyy"
```

```javascript JavaScript
// By ID
const response = await fetch(
  'https://api.chidori.africa/enterprise-service/api/deliveries/get?id=abc123-def456-ghi789',
  {
    headers: {
      'X-API-Key': 'sk_live_xxx.yyy'
    }
  }
);

const data = await response.json();
```

```python Python
import requests

# By tracking ID
response = requests.get(
    'https://api.chidori.africa/enterprise-service/api/deliveries/get',
    params={'trackingId': 'CHI-ABC123'},
    headers={'X-API-Key': 'sk_live_xxx.yyy'}
)

data = response.json()
```

## Delivery statuses

| Status | Description |
|--------|-------------|
| `PENDING` | Delivery created, awaiting payment |
| `PAID` | Payment received, awaiting driver assignment |
| `ASSIGNED` | Driver assigned to the delivery |
| `PICKED_UP` | Driver has picked up the package |
| `IN_TRANSIT` | Package is on the way to destination |
| `DELIVERED` | Package successfully delivered |
| `CANCELLED` | Delivery was cancelled |

## Use cases

<AccordionGroup>
  <Accordion title="Customer tracking page">
    Use the `trackingId` to let customers check their delivery status:
    
    ```javascript
    async function getDeliveryStatus(trackingId) {
      const response = await fetch(
        `https://api.chidori.africa/enterprise-service/api/deliveries/get?trackingId=${trackingId}`,
        { headers: { 'X-API-Key': apiKey } }
      );
      const { data } = await response.json();
      return {
        status: data.deliveryStatus,
        driver: data.driver,
        estimatedTime: data.estimatedDeliveryTime
      };
    }
    ```
  </Accordion>
  
  <Accordion title="Order management system">
    Use the delivery `id` to sync with your internal order system:
    
    ```javascript
    async function syncDeliveryStatus(deliveryId) {
      const response = await fetch(
        `https://api.chidori.africa/enterprise-service/api/deliveries/get?id=${deliveryId}`,
        { headers: { 'X-API-Key': apiKey } }
      );
      const { data } = await response.json();
      
      await updateOrderInDatabase({
        deliveryStatus: data.deliveryStatus,
        isPaid: data.isPaid,
        driverAssigned: !!data.driver
      });
    }
    ```
  </Accordion>
</AccordionGroup>
